Sentencia de bucle flexible

DESCRIPCIÓN

Esta sentencia permite crear un bucle de forma más flexible que los bucles característicos como WHILE o FROM .

Entre paréntesis deben especificarse tres expresiones diferentes (que pueden estar en blanco):

  • La primera expresión es típicamente una sentencia de asignación y se ejecuta una sóla vez, antes de empezar el bucle. Si está en blanco, no se ejecuta nada en especial. Se suele usar para preparar la variable que se usará como contador del bucle.
  • La segunda expresión es la condición del bucle y se evalúa tanto al principio, antes de la primera ejecución, como después de acabar cada ejecución posterior del bucle. Cuando esta condición se evalúe a falsa, el bucle se cortará inmediatamente y el control pasará a la siguiente sentencia después del END correspondiente. Si la segunda expresión está en blanco, se considerará siempre CIERTA.
  • La tercera expresión suele ser un incremento (empleando operadores como += o ++) y se ejecuta después de cada iteración del bucle, y antes de volver a comprobar la condición. Se usa para actualizar la variable de contador al siguiente valor.

Mediante un FOR es posible duplicar el comportamiento de un bucle WHILE o FROM, y también crear bucles más sofisticados. Aunque cualquier cosa hecha con FOR sería posible también hacerla con WHILE, resulta más claro y cómodo de cara a usar CONTINUE .

NOTAS
En Fenix, cualquier valor entero impar se considera cierto (como por ejemplo, 1) mientras cualquier valor par se considera falso (incluyendo el 0). Los operadores de comparación como == o > devuelven 1 ó 0 siempre, pero no hay razón para no poder usar cualquier otro cálculo.
