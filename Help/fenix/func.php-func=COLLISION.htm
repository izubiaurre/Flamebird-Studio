COLLISION ( ID )

Busca colisiones de cualquier tipo contra el proceso actual

PARÁMETROS
INTID : Identificador de un proceso, o de un tipo de proceso mediante la instrucción
DESCRIPCIÓN

Esta es la función principal de colisión, que busca un proceso que, según su gráfico, posición en pantalla según sus variables X e Y , y valores de flags, colisionaría con el actual en caso de ser ambos dibujados inmediatamente. En el caso de encontrar un proceso, esta función devuelve su ID (que siempre es un número impar). En caso contrario, devolverá 0.

Según el valor especificado como parámetro, es posible:

  • Encontrar cualquier proceso que haga colisión (pasando 0 como parámetro)
  • Encontrar cualquier proceso de un tipo determinado que haga colisión (para ello se pasa como parámetro la expresión TYPE PROCESO , donde PROCESO es el nombre de algún tipo de proceso
  • Comprobar si un proceso determinado hace colisión con el actual (pasando como parámetro su valor de ID , sea mediante el operador . o bien por estar almacenado en una variable como SON .
  • Comprobar si el ratón está encima del proceso actual, usando la expresión especial TYPE MOUSE .

La comprobación es perfecta a nivel de pixel, y tiene en cuenta diferencias de tamaño, rotaciones, y funciona incluso aunque los objetos no estén dentro de los límites visibles de la pantalla o aunque sólo la parte que colisiona esté fuera de pantalla. Esto tiene importantes consecuencias: por ejemplo, un objeto con forma de donut no colisionará con un objeto más pequeño que se dibuje en su interior, si este otro objeto no toca un pixel no transparente.

Sin embargo, ambos procesos deben estar visibles para que esta función compruebe la existencia de una colisión. Consulta la función SIGNAL para más información.

Las sucesivas llamadas a la función COLLISION con los mismos parámetros devuelven los siguientes procesos del tipo especificado que hacen colisión, hasta que ya no queda ninguno, en cuyo caso esta función devuelve 0. Este mecanismo permite, mediante un bucle, buscar todos los procesos que hacen colisión con el actual.

NOTAS
El funcionamiento de COLLISION(TYPE MOUSE) es un poco especial, ya que no se comprueba colisión con todo el gráfico del ratón (MOUSE.GRAPH), sino sólo con el centro indicado por MOUSE.X y MOUSE.Y, y la función devuelve 1 en caso de haberlo.
