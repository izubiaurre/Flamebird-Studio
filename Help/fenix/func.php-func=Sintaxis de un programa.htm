Proceso principal

Un programa se compone de datos y procesos

DESCRIPCIÓN

Un programa en Fenix se compone de varias secciones diferentes que pueden aparecer en cualquier orden. Cada sección comienza con una palabra clave que la identifica (como por ejemplo GLOBAL o ) y finaliza con la palabra clave END . Entre sección y sección no puede aparecer nada, salvo comentarios , directivas del preprocesador (ambas cosas pueden aparecer también en mitad de una sección) o la directiva especial INCLUDE .

Datos
Un programa dispone, por una parte, de datos tales como la puntuación, el número de vidas y cualquier otro tipo de información susceptible de variar durante el juego. Consulta Declaraciones de datos para obtener más información acerca de las secciones de datos disponibles en Fenix y cómo escribirlas en tu programa.
Código
Por otra parte, el código del programa se distribuye en procesos que indican el funcionamiento del juego: qué objetos aparecen en pantalla, y cómo deben reaccionar ante las acciones del jugador o entre ellos mismos. Consulta Declaraciones de procesos para

Se puede decir que los datos indican el estado del juego, mientras el código muestra las reglas del mismo.

La única sección obligatoria en todo programa en Fenix es el proceso principal . Este proceso contiene el código que se ejecuta al principio del juego, cuando el usuario lo inicia por primera vez. Normalmente el código del programa se distribuye en otros procesos que contienen código especializado en objetos concretos que aparecen en pantalla o en acciones concretas (como por ejemplo, mostrar un menú de opciones). El proceso principal se encarga de crear y ejecutar el resto de procesos según sea necesario desde un principio.
