INTEGER WINDOW ( [ TÍTULO ], [ ESTILO ] )

Crea una nueva ventana

PARÁMETROS
INT TÍTULO : (opcional)   Título de la ventana
INT ESTILO : (opcional)   Estilo de la ventana
VALOR DE RETORNO

INTEGER   :   Identificador único de la ventana

DESCRIPCIÓN

Crea una nueva ventana en memoria. Una ventana es una sección de pantalla delimitada por un rectángulo que puede contener uno o más controles. Una ventana puede disponer de una barra de título, un borde fijo o redimensionable, botones del sistema, o ser desplazada libremente por el usuario. El comportamiento exacto de la ventana se define mediante el estilo de la misma, que puede especificarse como parámetro a esta función y puede ser uno de los siguientes:

  • STYLE_FIXED: La ventana cuenta con un borde, pero no puede redimensionarse. Usa este estilo para ventanas que muestran controles que no vayan a redimensionarse, como por ejemplo una caja de herramientas, pero que formen parte del interfaz de la aplicación.
  • STYLE_STATIC: La ventana no tiene borde, y no puede ser movida ni redimensionada por el usuario. Usa esta ventana cuando quieras crear efectos especiales o disponer de elementos en el interfaz de usuario que quieras manipular manualmente en tu programa.
  • STYLE_DIALOG: La ventana cuenta con un borde y no puede redimensionarse ni ocultarse (minimizarse). Usa estas ventanas para obtener información puntual del usuario (son llamadas cuadros de diálogo).
  • STYLE_RESIZABLE_DIALOG: Ventana idéntica a la anterior, pero en este caso el usuario puede redimensionarla. Algunos cuadros de diálogo pueden ser útiles a mayor tamaño, por ejemplo aquellos que contengan listas de información que puedan crecer hasta un tamaño indeterminado.
  • STYLE_NORMAL: Ventana corriente, que puede minimizarse, redimensionarse y maximizarse. Este es el estilo por defecto si no empleas ninguno.
  • STYLE_DOCK_NORTH: Esta ventana se posicionará automáticamente en la parte superior de la pantalla, abarcando todo su ancho, y siempre será visible por encima del resto. Además, el usuario no podrá moverla de sitio. Usa este estilo para crear barras de menús o herramientas.
  • STYLE_DOCK_SOUTH: Idéntico al anterior, pero para la parte inferior de la pantalla. Usalo para crear barras de tareas o de estado.
  • STYLE_DOCK_WEST: Esta ventana se posicionará a la izquierda del escritorio, siempre visible por encima del resto.
  • STYLE_DOCK_EAST: Idéntico al anterior, pero para la derecha. Un punto importante en ambos estilos es que los estilos DOCK_NORTH y DOCK_SOUTH tienen preferencia, es decir, la barra de menús o de tareas siempre tocará los extremos derechos e izquierdos de pantalla.

Además, esta función permite especificar un título. Sólo las ventanas que disponen de título mostrarán la cabecera de la ventana con los botones de cerrar, minimizar y maximizar. Si se desea, es posible crear una ventana con título "oculto", es decir, una ventana con título (que aparece en la lista de tareas) pero sin cabecera. Para ello es preciso usar un título con prefijo "*". Una ventana titulada "*Tools" aparecerá con el texto "Tools" en la barra de tareas, pero no tendrá cabecera.

Todas las ventanas, excepto las de estilos DOCK y FIXED, se dibujarán con una sombra para destacar su profundidad.

Las ventanas de estilos DOCK son especiales, ya que una ventana maximizada se redimensionará de manera que ocupe el espacio que estas dejen disponible, y no más.

La ventana recién creada no estará inmediatamente visible, pero puede agregarse al escritorio en cualquier momento mediante la función DESKTOP_ADDWINDOW . Es buena idea crear primero los controles que vaya a contener la ventana mediante repetidas llamadas a la función WINDOW_ADDCONTROL .

Esta función devuelve un valor entero que identifica de forma única a la ventana creada. Este valor debe guardarse y utilizar en cualquier llamada posterior al resto de funciones con prefijo WINDOW_.
