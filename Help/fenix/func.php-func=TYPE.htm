Declara un tipo de dato definido por el usuario

DESCRIPCIÓN

Un bloque TYPE debe situarse fuera de cualquier sección de datos o proceso. TYPE declara un tipo de dato definido por el usuario y compuesto a su vez por varios datos de otro tipo. TYPE no crea ninguna variable concreta en el momento en el que aparece: simplemente, le otorga un significado al identificador utilizado y permite que sea usado posteriormente en cualquier sección de datos.

A partir de la aparición de TYPE, es posible utilizar el identificador indicado para crear estructuras cuyo contenido será exactamente igual al indicado en el interior del bloque TYPE, pero sin tener que declarar de nuevo todas las variables en un nuevo bloque. Si, por ejemplo, al principio del programa aparece este código: 

 TYPE color 

 BYTE r, g, b; 

 END
Sería posible declarar una o más variables del nuevo tipo "color" en cualquier sección de datos: 
 GLOBAL 

 color p; 

 END
En este ejemplo, la variable "p" es una estructura con tres componentes de tipo byte (r, g, b).

Es posible usar valores por defecto en el TYPE mediante la sintaxis habitual (= valor). Estos valores serán utilizados por todas las variables del nuevo tipo que no sean inicializadas manualmente.

Ventajas de TYPE

Aunque es posible usar STRUCT para crear una variable de tipo estructura de la misma forma que con TYPE, TYPE es mucho más cómodo si se tienen que crear múltiples variables en lugares diferentes del programa con la misma estructura. Además, TYPE permite crear punteros a estructuras. Siguiendo el ejemplo anterior, podríamos crear una variable como: 

 color pointer ptr;
y de esta manera acceder a estructuras remotas, pasar la dirección de una estructura como parámetro a un proceso y en general todas las facilidades avanzadas que permite un puntero.
