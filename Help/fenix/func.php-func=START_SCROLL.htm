START_SCROLL ( NÚMERO, FICHERO, GRÁFICO, FONDO, REGION, FLAGS )

Crea una zona de scroll

PARÁMETROS
INTNÚMERO : Número del scroll (0 a 9)
INTFICHERO : Número de fichero FPG
INTGRÁFICO : Número de gráfico para el primer plano del scroll
INTFONDO : Número de gráfico para el fondo del scroll, 0 para ninguno
INTREGION : Número de región a ocupar
INTFLAGS : Flags
DESCRIPCIÓN

Inicializa y activa una ventana de scroll. Las ventanas de scroll permiten mostrar en pantalla una zona de juego más grande que la pantalla, de manera que sólo es posible ver una parte del total. Los procesos pueden configurarse para ser mostrados en el interior de esa zona de juego, en lugar de la pantalla normal. Es posible desplazar manualmente la posición de la zona visible o hacer que ésta persiga automáticamente a algún proceso importante.

Para poder usar una zona de scroll, primero hay que activarla con esta función, que establece los parámetros básicos de la misma. Luego es posible ajustarlos modificando la estructura global SCROLL .

Toda zona de scroll debe disponer de un gráfico de primer plano que debe ser válido, cuyo código y fichero FPG se le pasa a esta función como parámetro. Se considera un error grave no usar un gráfico válido. El scroll puede contar con un gráfico de segundo plano o fondo, que se verá a través de los pixels transparentes del gráfico de primer plano. Este fondo es opcional y es posible no utilizarlo pasando el valor 0 como parámetro "fondo".

Una zona de scroll puede ocupar toda la pantalla si se le pasa 0 como región a esta función, o bien sólo una parte, usando el número de una región válida. Véase DEFINE_REGION .

La zona de scroll definida cuenta con varias opciones o "flags", que pueden ser 0 para el funcionamiento normal o la suma de una o más de las siguientes opciones:

  • 1 : Mosaico horizontal del primer plano
  • 2 : Mosaico vertical del primer plano
  • 4 : Mosaico horizontal del fondo
  • 8 : Mosaico vertical del fondo

Por defecto, el scroll está delimitado por el tamaño del primer plano, lo que significa que la posición visible nunca se desplazará más allá de sus límites. Sin embargo, se puede activar un modo de "mosaico" que hace que el gráfico de primer plano se repita, como un gráfico de fondo en un escritorio. Con este modo activado, no hay límite para el movimiento. Es posible eliminar el límite para el movimiento vertical u horizontal, escogiendo entre los flags anteriores. De la misma forma, puede activarse un modo de mosaico para el fondo, si bien eso no provoca efectos en las limitaciones de movimiento para el scroll.

Por defecto, los procesos no son dibujados en la zona de scroll. Para ello, es preciso que la variable local del proceso CTYPE sea igual a la constante predefinida C_SCROLL (con valor 1). Hecho este cambio, las variables de posición X e Y del proceso se considerarán relativas a la zona de scroll y no a la pantalla.

Es posible activar hasta 10 scrolls a la vez. Esto puede ser útil para mostrar dos zonas de scroll distintas, pero el uso más común es mostrar dos ventanas de movimiento dentro de la misma zona (por ejemplo para partidas a dos jugadores). Por defecto, los procesos con CTYPE 1 se mostrarán en todas las zonas de scroll. Para escoger las zonas concretas en las que un proceso es visible, hay que asignar a su variable local CNUMBER la suma de una o más de las constantes predefinidas C_0, C_1 a C_9, para indicar qué números de scrolls son aquellos en los que el proceso será visible. Así, con CNUMBER = C_0 + C_2 se hace que un proceso sea visible en las zonas de scroll 0 y 2, pero no en la 1 ni en ninguna otra (si existe).

Normalmente siempre es conveniente modificar los valores de la estructura global SCROLL , que contiene opciones de configuración adicionales para cada zona de scroll. Es posible modificar los valores de esta estructura cada frame si es preciso, mientras que la configuración especificada por esta función es fija para cada zona hasta que se detenga su funcionamiento con STOP_SCROLL .
